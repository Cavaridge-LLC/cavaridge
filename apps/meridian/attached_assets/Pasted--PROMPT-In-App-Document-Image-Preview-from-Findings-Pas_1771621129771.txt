# PROMPT: In-App Document & Image Preview from Findings

> **Paste into Replit Agent.** Adds an inline preview panel so users can view
> source documents and images directly in the interface — no download required.
> Works from findings, the document list, and Ask MERIDIAN citations.

---

```
Add in-app document and image preview to MERIDIAN. When a finding has an 
associated document or image, users should be able to view it inline — 
rendered beautifully in the interface, not downloaded.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PART 1 — PREVIEW API ENDPOINT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Create GET /api/documents/[id]/preview that returns previewable content 
based on file type.

For IMAGES (png, jpg, jpeg, gif, webp, tiff, svg):
  - Read the file from storage
  - Return with correct Content-Type header (image/png, image/jpeg, etc.)
  - Support a query param ?size=thumb|medium|full
    - thumb: max 200px longest side (for list thumbnails)
    - medium: max 800px longest side (for preview panel)
    - full: original size (for zoom/lightbox)
  - Use the "sharp" npm package to resize on-the-fly (cache the result)
  - Return the image binary directly so it can be used as an <img src="">

For PDFs:
  - Return the file with Content-Type: application/pdf
  - The frontend will render it in an <iframe> or with a PDF viewer library

For TEXT-BASED files (md, txt, log, cfg, conf, json, xml, yaml, yml, csv, 
tsv, html, css, js, ts, py, sh, sql):
  - Return JSON: { content: "file text content", language: "json" }
  - The frontend renders with syntax highlighting

For DOCX:
  - Use mammoth to convert to HTML
  - Return JSON: { html: "<p>converted content...</p>" }
  - The frontend renders the HTML in a styled container

For XLSX:
  - Use SheetJS to convert to HTML tables
  - Return JSON: { html: "<table>...</table>", sheets: ["Sheet1", "Sheet2"] }

For PPTX:
  - Return the extracted slide text formatted as sections
  - Return JSON: { slides: [{ number: 1, text: "..." }, ...] }

For EMAIL (.eml):
  - Return JSON: { from, to, subject, date, body_html, body_text }

For ALL types, also return metadata:
{
  "id": "...",
  "filename": "...",
  "file_type": "png",
  "file_size": 234567,
  "classification": "Security Assessment",
  "extraction_status": "extracted",
  "preview_type": "image|pdf|text|html|slides|email",
  "extracted_text": "...",  
  "vision_analysis": { ... },  // if image with vision results
  "metadata_json": { ... }
}

Security: enforce tenant isolation. The requesting user must have access 
to the deal this document belongs to.

Build a reusable <DocumentPreview> React component that renders any 
document type inline.

Props: { documentId, onClose, defaultTab? }

Layout: a slide-over panel that opens from the right side of the screen

HEADER:
- Back arrow (closes panel)
- Filename (truncated if long, full on hover tooltip)
- Classification badge (colored by category)

TABS:

TAB 1 — "Preview" (default):
  Renders the document content based on type:

  FOR IMAGES:
  - Render the image centered in the panel using the medium size
  - Enable zoom: click to toggle between fit-to-panel and full-size
  - Add zoom controls: + / - buttons and a "Fit" button
  - Mouse wheel scrolls zoom level
  - If image has vision analysis, show a semi-transparent overlay button:
    "👁 View AI Analysis" that toggles an overlay with the vision 
    description and detected findings highlighted

  FOR PDFs:
  - Render in an <iframe> with the PDF URL from the preview endpoint
  - Or use react-pdf (install @react-pdf/renderer or react-pdf) for 
    a richer experience with page navigation:
    - Page counter: "Page 3 of 12"
    - Previous/Next page buttons
    - Zoom controls
    - Fits width of panel by default

  FOR TEXT/CODE:
  - Render in a code block with syntax highlighting
  - Use a syntax highlighting library (prism-react-renderer or 
    react-syntax-highlighter — install whichever works best)
  - Auto-detect language from file extension:
    .json → json, .xml → xml, .yaml/.yml → yaml, .py → python,
    .js → javascript, .ts → typescript, .sql → sql, .sh → bash,
    .csv → csv, .md → markdown, .html → html, .css → css
  - Show line numbers
  - Monospace font (JetBrains Mono)
  - Dark panel background (var(--bg-panel)) regardless of theme

  FOR DOCX/HTML:
  - Render the converted HTML in a styled container
  - Apply clean typography: readable font, proper heading sizes, 
    table borders, list styles
  - Scope the CSS so the document HTML doesn't bleed into the app

  FOR XLSX:
  - Render as HTML tables with tabs for each sheet
  - Alternating row colors for readability
  - Horizontal scroll if table is wide
  - Sheet selector tabs above the table

  FOR PPTX:
  - Render each slide as a card in a vertical scroll
  - Slide number badge on each card
  - Slide text content with basic formatting

  FOR EMAIL:
  - Render like an email client:
    From: / To: / Date: / Subject: headers
    Horizontal rule
    Body content (HTML rendered, or plain text)
    Attachment list (names only, with file type icons)

  FOR UNSUPPORTED TYPES:
  - Show the extracted text if available
  - Otherwise show: "Preview not available for this file type. 
    Extracted text is shown in the 'Extracted Text' tab."

TAB 2 — "Details":
  Show document metadata in a clean key-value layout:
  - Filename
  - File type
  - File size (formatted: "22.7 KB")
  - Classification (with badge)
  - Extraction status (with badge)
  - Upload date
  - Uploaded by (user name)
  - Deal name
  - Content hash (truncated, copy button)
  - Text length (formatted: "23,227 characters")
  - Chunk count
  - Embedding status (embedded / indexed / none)
  - Parent archive (if extracted from ZIP, show link to parent)
  - Folder path (if from ZIP)

  If image with vision analysis:
  - "AI Vision Analysis" section
  - Full description from the vision model
  - Extracted text from the image
  - Confidence indicators if available

TAB 3 — "Extracted Text":
  Show the full extracted_text from the documents table in a scrollable 
  monospace text area. This is the raw text the system extracted — useful 
  for seeing exactly what MERIDIAN "read" from the document.
  
  - Monospace font (JetBrains Mono)
  - Line numbers
  - Search within text (Ctrl+F or a search bar at the top)
  - "Copy All" button to copy the full text to clipboard
  - If no extracted text: "No text was extracted from this document."

TAB 4 — "Findings" (only show this tab if findings exist for this document):
  List all findings that were generated from or linked to this document.
  Show each finding as a card:
  - Severity badge (critical=red, high=amber, medium=yellow, low=gray)
  - Title
  - Description (truncated, expandable)
  - Pillar name badge
  - Status badge (open/acknowledged/remediated)
  - "[Auto-detected from image analysis]" tag if applicable
  - "View in Risk Engine" link that navigates to the Risk view and

PART 7 — CACHING & PERFORMANCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Previewing files on every click will be slow without caching:

1. IMAGE THUMBNAILS:
   - Generate thumb and medium sizes on first request
   - Cache to disk: /cache/thumbnails/{document_id}_{size}.jpg
   - Serve from cache on subsequent requests
   - Set Cache-Control: max-age=86400 (24 hours) on the response
   - Clear cache when a document is deleted

2. DOCX → HTML CONVERSION:
   - Cache the converted HTML in the document's metadata_json 
     under key "preview_html"
   - Only reconvert if the document changes (compare content_hash)

3. XLSX → HTML TABLE:
   - Same caching strategy — store in metadata_json under "preview_tables"

4. PDF:
   - Serve the original file directly (no conversion needed)
   - Set Content-Disposition: inline (not attachment) so the browser 
     renders it instead of downloading

5. LAZY LOADING:
   - Thumbnails in the document list and finding source badges should 
     use loading="lazy" on the <img> tag
   - Only fetch the preview data when the user clicks to open the panel 
     (not on page load)
```
