2.3 — Add rate limiting middleware

Install express-rate-limit and create server/middleware/rate-limit.ts.
Define three rate limit tiers:

Global default: 100 requests per minute per IP. Apply to all routes.
Auth strict: 5 requests per minute per IP. Apply to /api/auth/login and /api/auth/register.
AI endpoints: 20 requests per minute per IP. Apply to all routes containing /qa/, /export/, /simulate/, /classify, /embed, /infra-analysis, /playbook.

In server/index.ts, apply the global limiter as middleware before routes. Apply the stricter limiters to their specific route paths.
Return standard 429 responses with a JSON body: { error: "Too many requests", retryAfter: <seconds> }.
Include X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset headers on all responses.
Do not rate-limit the /api/version or /api/system-status health check endpoints.


2.4 — Add CSRF protection

Implement double-submit cookie CSRF protection.
Create server/middleware/csrf.ts:

On every response, set a XSRF-TOKEN cookie (httpOnly: false, sameSite: 'strict', secure: true in production).
On every state-changing request (POST, PUT, PATCH, DELETE), validate that the X-XSRF-TOKEN header matches the cookie value.
Exempt these paths from CSRF validation: /api/auth/login, /api/auth/register, /api/auth/logout, /api/account-requests (public form submission), and any webhook endpoints.

On the frontend, add an Axios/fetch interceptor (or equivalent in the existing API client) that reads the XSRF-TOKEN cookie and attaches it as the X-XSRF-TOKEN header on all requests.
Return 403 with { error: "CSRF token invalid" } on mismatch.
Apply in server/index.ts after session middleware but before routes.

2.8 — Add structured logging

Create server/logger.ts using pino (install it).
Configure:

JSON output format
Log levels: trace, debug, info, warn, error, fatal
Default level: info in production, debug in development
Base context fields: app: "CVG-MERIDIAN", version from version.json

Create an Express middleware in the same file that attaches a child logger to each request with: requestId (generate a UUID), userId (from session if authenticated), orgId (from session if authenticated), method, path.
Replace all console.log and console.error calls across the server codebase with the appropriate logger method:

console.log → logger.info
console.error → logger.error
console.warn → logger.warn

In route handlers, use req.log (the request-scoped child logger) instead of the global logger so every log line includes request context.
Do NOT change any business logic — only swap the logging calls.
