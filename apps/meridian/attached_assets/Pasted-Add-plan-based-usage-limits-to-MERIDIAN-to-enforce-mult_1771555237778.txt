Add plan-based usage limits to MERIDIAN to enforce multi-tenant fairness and prepare for monetization.

ENFORCEMENT:

Create a server-side utility "checkPlanLimit" that:
1. Accepts: (organizationId, limitType) where limitType is "users", "deals", "storage", "documents", "queries", "baselines"
2. Queries current usage from the database
3. Compares against the org's plan_tier limits
4. Returns: { allowed: boolean, current: number, limit: number, limitType: string }

Apply limit checks at these points:
- Creating a user/invitation → check "users" limit
- Creating a deal → check "deals" limit
- Uploading a document → check "storage" limit (sum file sizes) and "documents" limit
- Sending an Ask MERIDIAN query → check "queries" limit

Create a table "usage_tracking":
- id (UUID, primary key)
- organization_id (UUID, foreign key)
- metric (text: "chat_queries", "documents_uploaded", "storage_bytes")
- period (text: "2026-02" for monthly metrics)
- count (integer)
- updated_at (timestamp)

Increment usage_tracking when actions occur. Reset monthly counts on the 1st of each month (or check against the period field).

LIMIT HIT UI:
When a user tries to do something beyond their plan limit, show a styled modal:
- "Plan Limit Reached" header
- "Your [Starter/Professional] plan allows [X] [users/deals/etc]. You're currently using [Y]."
- "Upgrade to [next tier] to get [higher limit]."
- "Contact sales@meridian-platform.com" button
