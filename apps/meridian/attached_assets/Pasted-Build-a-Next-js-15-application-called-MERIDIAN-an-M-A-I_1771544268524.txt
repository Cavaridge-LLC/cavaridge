DATABASE SCHEMA — Create these PostgreSQL tables:

1. "users" table:
   - id (UUID, primary key)
   - email (text, unique)
   - name (text)
   - role (text: "admin", "analyst", "viewer")
   - organization_id (UUID, foreign key)
   - created_at (timestamp)

2. "organizations" table:
   - id (UUID, primary key)
   - name (text)
   - industry_default (text)
   - created_at (timestamp)

3. "deals" table:
   - id (UUID, primary key)
   - organization_id (UUID, foreign key)
   - deal_code (text, unique, like "MRD-2026-017")
   - target_name (text)
   - industry (text: "Healthcare", "Financial Services", "Manufacturing", "Technology/SaaS", "Retail", "Professional Services")
   - stage (text: "Preliminary Review", "Data Room Open", "Active Diligence", "Integration Planning", "Closed")
   - status (text: "ahead", "on-track", "at-risk", "blocked")
   - facility_count (integer)
   - user_count (integer)
   - estimated_integration_cost (text)
   - composite_score (decimal)
   - documents_uploaded (integer, default 0)
   - documents_analyzed (integer, default 0)
   - created_at (timestamp)
   - updated_at (timestamp)

4. "pillars" table:
   - id (UUID, primary key)
   - deal_id (UUID, foreign key)
   - pillar_name (text: "Infrastructure & Architecture", "Cybersecurity Posture", "Regulatory Compliance", "Integration Complexity", "Technology Org & Talent", "Data Assets & Governance")
   - score (decimal, 1.0-5.0)
   - weight (decimal, 0.0-1.0)
   - finding_count (integer)
   - updated_at (timestamp)

5. "findings" table:
   - id (UUID, primary key)
   - deal_id (UUID, foreign key)
   - pillar_id (UUID, foreign key)
   - severity (text: "critical", "high", "medium", "low")
   - title (text)
   - description (text)
   - impact_estimate (text)
   - remediation_notes (text)
   - source_documents (text array)
   - source_count (integer)
   - status (text: "open", "acknowledged", "remediated")
   - created_at (timestamp)

6. "documents" table:
   - id (UUID, primary key)
   - deal_id (UUID, foreign key)
   - filename (text)
   - file_type (text)
   - file_size (integer)
   - classification (text)
   - upload_status (text: "uploaded", "processing", "analyzed", "error")
   - page_count (integer)
   - uploaded_by (UUID, foreign key to users)
   - created_at (timestamp)

7. "chat_messages" table:
   - id (UUID, primary key)
   - deal_id (UUID, foreign key)
   - user_id (UUID, foreign key)
   - role (text: "user", "assistant")
   - content (text)
   - confidence_score (decimal, nullable)
   - source_count (integer, nullable)
   - model_used (text, nullable)
   - created_at (timestamp)

8. "baseline_profiles" table:
   - id (UUID, primary key)
   - organization_id (UUID, foreign key)
   - name (text)
   - profile_data (JSONB — stores baseline standards per category)
   - is_default (boolean)
   - created_at (timestamp)
