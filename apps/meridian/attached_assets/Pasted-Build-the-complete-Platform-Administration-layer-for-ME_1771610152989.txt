PART A — ROLE HIERARCHY & DATABASE CHANGES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NEW 7-TIER ROLE HIERARCHY (top to bottom):

1. platform_owner — God-mode. Sees all orgs, manages everything. Only 1-2 people.
2. platform_admin — Cavaridge staff. Full read access, manages orgs/users, 
   cannot delete orgs or change billing.
3. org_owner — Owner of a single tenant org. Full control within their org only.
4. org_admin — Admin within a specific org. Manages users and deals for that org.
5. analyst — Creates and works on deals within their org.
6. integration_pm — Works on assigned deals only.
7. viewer — Read-only access to assigned deals.

DATABASE CHANGES:

1. UPDATE "users" table:
   - Change the "role" column to support all 7 values above
   - Add column: is_platform_user (boolean, default false)
   - Platform users (platform_owner, platform_admin) have is_platform_user = true.
     Their organization_id points to the Cavaridge org, but they can access ALL orgs.

2. CREATE TABLE "platform_settings":
   - id (UUID, primary key)
   - setting_key (text, unique)
   - setting_value (JSONB)
   - updated_by (UUID, foreign key to users)
   - updated_at (timestamp)
   
   Seed with these rows:
   - "platform_name": "MERIDIAN"
   - "platform_version": "2.0.0"
   - "registration_mode": "request"  (values: "open", "request", "closed")
   - "default_plan_tier": "starter"
   - "maintenance_mode": false
   - "allowed_email_domains": []

3. CREATE TABLE "account_requests":
   - id (UUID, primary key)
   - company_name (text)
   - contact_name (text)
   - contact_email (text)
   - contact_phone (text, nullable)
   - industry (text)
   - estimated_deals_per_year (integer, nullable)
   - estimated_users (integer, nullable)
   - message (text, nullable)
   - status (text: "pending", "approved", "rejected", "converted")
   - reviewed_by (UUID, nullable, foreign key to users)
   - reviewed_at (timestamp, nullable)
   - review_notes (text, nullable)
   - organization_id (UUID, nullable — set when converted to an actual org)
   - created_at (timestamp)
