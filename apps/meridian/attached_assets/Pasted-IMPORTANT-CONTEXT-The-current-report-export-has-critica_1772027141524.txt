PDF RENDERING FIXES — Technical requirements
═══════════════════════════════════════════════════════════════════════════════════════

Whatever PDF library you're using (jsPDF, pdfmake, puppeteer, react-pdf, etc.),
apply these fixes:

1. TABLE ROW KEEPING:
   Every table row must stay together on one page. Never split a row.
   If using pdfmake: set `dontBreakRows: true` on every table.
   If using puppeteer/HTML: add CSS `tr { break-inside: avoid; }`.
   If using jsPDF-autotable: set `rowPageBreak: 'avoid'`.

2. REPEATING HEADERS:
   When a table spans multiple pages, repeat the header row.
   If using pdfmake: set `headerRows: 1` on the table.
   If using puppeteer/HTML: use `<thead>` tags.
   If using jsPDF-autotable: this is automatic.

3. NO GHOST PAGES:
   After the last content element, DO NOT render any more pages.
   Check for:
   - Empty page breaks at the end of sections
   - Footer-only pages being generated
   - Off-by-one errors in page counting loops
   - CSS `page-break-after: always` on the last section

4. PAGE NUMBERS:
   Footer format: "MERIDIAN — Confidential — Page X of Y"
   Y must equal the actual number of content pages, not some
   pre-calculated estimate.

5. TEXT TRUNCATION:
   Long finding titles and descriptions must be truncated to fit
   table cells. Use "..." for overflow. Never let text push a
   cell to be taller than ~3 lines in a table.

6. CONSISTENT FONTS:
   Use one font family throughout. Recommended:
   - Headers: Bold, 11-14pt
   - Body: Regular, 10pt
   - Tables: Regular, 9pt
   - Footnotes: Regular, 8pt

7. COLOR CODING:
   Severity badges should use consistent colors:
   - CRITICAL: Red (#DC2626)
   - HIGH: Orange (#EA580C)
   - MEDIUM: Amber (#D97706)
   - LOW: Blue (#2563EB)
   - Score risk levels follow same scheme
