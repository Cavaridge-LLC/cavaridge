CREATE TABLE document_classifications (
  id SERIAL PRIMARY KEY,
  document_id INTEGER NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
  deal_id INTEGER NOT NULL REFERENCES deals(id) ON DELETE CASCADE,
  tenant_id INTEGER NOT NULL,
  
  -- Classification fields
  document_type VARCHAR(100) NOT NULL,
  -- Allowed types: 'network_diagram', 'security_policy', 'backup_config',
  -- 'vendor_contract', 'org_chart', 'asset_inventory', 'firewall_rules',
  -- 'compliance_report', 'disaster_recovery_plan', 'cloud_config',
  -- 'email_migration', 'software_license', 'sla_agreement',
  -- 'penetration_test', 'vulnerability_scan', 'configuration_export',
  -- 'screenshot', 'invoice', 'proposal', 'meeting_notes', 'unknown'
  
  -- Pillar mapping (which MERIDIAN pillars does this document inform?)
  pillar_infrastructure BOOLEAN DEFAULT FALSE,
  pillar_security BOOLEAN DEFAULT FALSE,
  pillar_operations BOOLEAN DEFAULT FALSE,
  pillar_compliance BOOLEAN DEFAULT FALSE,
  pillar_scalability BOOLEAN DEFAULT FALSE,
  pillar_strategy BOOLEAN DEFAULT FALSE,
  
  confidence DECIMAL(3,2) DEFAULT 0.00,  -- 0.00 to 1.00
  classification_reasoning TEXT,
  
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_doc_class_deal ON document_classifications(deal_id);
CREATE INDEX idx_doc_class_type ON document_classifications(document_type);

This function takes extracted text (first 3000 chars) + filename + file type
and sends it to Claude to classify. Here is the EXACT prompt to use:

---
SYSTEM PROMPT FOR CLASSIFICATION:
"You are a document classifier for IT due diligence assessments. Given a 
document's filename and content excerpt, classify it and map it to assessment
pillars.

Respond in JSON only, no markdown, no explanation:
{
  "document_type": "<one of the allowed types>",
  "pillars": {
    "infrastructure": true/false,
    "security": true/false,
    "operations": true/false,
    "compliance": true/false,
    "scalability": true/false,
    "strategy": true/false
  },
  "confidence": 0.0-1.0,
  "reasoning": "<one sentence explaining classification>"
}

Allowed document_type values:
network_diagram, security_policy, backup_config, vendor_contract, org_chart,
asset_inventory, firewall_rules, compliance_report, disaster_recovery_plan,
cloud_config, email_migration, software_license, sla_agreement,
penetration_test, vulnerability_scan, configuration_export, screenshot,
invoice, proposal, meeting_notes, unknown

A document can map to MULTIPLE pillars. For example, a firewall config maps
to both infrastructure and security. A vendor SLA maps to operations and
compliance."
---

USER MESSAGE FORMAT:
"Filename: {filename}
File type: {mime_type}
Content excerpt:
{first 3000 characters of extracted text, or vision description for images}"

Use the model already configured in your AI orchestration layer (Claude via
litellm or direct Anthropic SDK). Use max_tokens: 500.

Parse the JSON response. If parsing fails, set document_type to 'unknown',
all pillars to false, confidence to 0.0.

This allows manual override of classification. Set confidence to 1.0 for
manual classifications (human verified).
