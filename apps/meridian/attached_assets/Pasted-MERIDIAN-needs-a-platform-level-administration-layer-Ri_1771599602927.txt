MERIDIAN needs a platform-level administration layer. Right now the highest role 
is "owner" scoped to a single organization. We need a tier ABOVE that — a 
Platform Owner who manages the entire MERIDIAN instance across all organizations.

NEW ROLE HIERARCHY (top to bottom):

1. PLATFORM OWNER — Cavaridge Holdings. God-mode. Sees everything, manages 
   everything, can impersonate any org. Only 1-2 people ever have this role.
2. PLATFORM ADMIN — Cavaridge staff who help manage the platform. Full read 
   access, can manage orgs and users, cannot delete orgs or change billing.
3. ORG OWNER — The owner of a specific tenant organization (e.g., a PE firm). 
   Full control within their org only.
4. ORG ADMIN — Admin within a specific org. Manages users and deals for their org.
5. ANALYST — Creates and works on deals within their org.
6. INTEGRATION PM — Works on assigned deals only.
7. VIEWER — Read-only access to assigned deals.

DATABASE CHANGES:

1. UPDATE "users" table:
   - Change the "role" column to support: "platform_owner", "platform_admin", 
     "org_owner", "org_admin", "analyst", "integration_pm", "viewer"
   - Add column: is_platform_user (boolean, default false)
   - Platform users (platform_owner, platform_admin) have is_platform_user = true 
     and their organization_id points to the Cavaridge org, but they can access 
     ALL organizations.

2. CREATE TABLE "platform_settings":
   - id (UUID, primary key)
   - setting_key (text, unique)
   - setting_value (JSONB)
   - updated_by (UUID, foreign key to users)
   - updated_at (timestamp)
   
   Seed with:
   - "platform_name": "MERIDIAN"
   - "platform_version": "2.0.0"
   - "registration_mode": "request" (values: "open", "request", "closed")
   - "default_plan_tier": "starter"
   - "maintenance_mode": false
   - "allowed_email_domains": [] (empty = allow all)

3. CREATE TABLE "account_requests":
   - id (UUID, primary key)
   - company_name (text)
   - contact_name (text)
   - contact_email (text)
   - contact_phone (text, nullable)
   - industry (text)
   - estimated_deals_per_year (integer, nullable)
   - estimated_users (integer, nullable)
   - message (text, nullable — "tell us about your use case")
   - status (text: "pending", "approved", "rejected", "converted")
   - reviewed_by (UUID, nullable, foreign key to users)
   - reviewed_at (timestamp, nullable)
   - review_notes (text, nullable — internal notes from platform admin)
   - organization_id (UUID, nullable — set when converted to an actual org)
   - created_at (timestamp)
