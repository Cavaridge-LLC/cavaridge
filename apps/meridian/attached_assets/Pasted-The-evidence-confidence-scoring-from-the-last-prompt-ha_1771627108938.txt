If there are dozens of "low" severity findings auto-generated from 
document analysis, the scoring system needs a DIMINISHING RETURNS model 
so that the 50th low-severity finding doesn't have the same impact as 
the 1st:

UPDATED SCORING with diminishing returns:

  let findingScore = 5.0;
  
  // Critical and high: full penalty each (these are always significant)
  findingScore -= critical * 0.8;
  findingScore -= high * 0.5;
  
  // Medium: diminishing returns after 5
  const mediumPenalty = Math.min(medium, 5) * 0.25 + 
                        Math.max(0, medium - 5) * 0.10;
  findingScore -= mediumPenalty;
  
  // Low: diminishing returns after 3
  const lowPenalty = Math.min(low, 3) * 0.1 + 
                     Math.max(0, low - 3) * 0.03;
  findingScore -= lowPenalty;
  
  // Floor
  findingScore = Math.max(1.0, findingScore);

This way:
  - 3 low findings = -0.30 (same as before)
  - 10 low findings = -0.30 + 0.21 = -0.51 (not -1.0)
  - 50 low findings = -0.30 + 1.41 = -1.71 (not -5.0)
  - Score can never drop below 1.0 regardless
