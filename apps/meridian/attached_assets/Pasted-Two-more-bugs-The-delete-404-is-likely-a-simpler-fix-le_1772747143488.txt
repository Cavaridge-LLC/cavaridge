**BUG 1: Batch delete returns 404 "Document not found"**

**Error:** `404: {"message":"Document not found"}` when deleting a document that is visibly present in the UI.

**Most likely causes (check in order):**

1. **Route parameter mismatch.** The batch delete endpoint may be sending document IDs in the request body, but the route handler may be looking for them as URL params. Check the frontend component that calls the delete — verify the API call format matches what the route expects.

2. **Org-scoping filter mismatch.** The delete handler in `server/storage.ts` or the route in `server/routes/documents.ts` queries the document with both `id` and `organizationId`. If the org ID isn't being passed correctly, or the join path through the `deals` table isn't matching, the lookup returns null → 404.

3. **Single vs. batch path.** The UI says "Delete 1 Documents" and the error says "Batch delete failed" — this is using the batch delete endpoint. Check if the batch delete route iterates over IDs and calls a single-document delete function that returns 404 on the first miss, aborting the entire batch. The single-document delete function may expect different parameters than what the batch path provides.

**Debugging steps:**
1. Open `server/routes/documents.ts` — find the batch delete endpoint (likely `DELETE /api/documents` or `DELETE /api/deals/:dealId/documents`)
2. Log the incoming document IDs and the authenticated user's `organizationId`
3. Run the same query manually against the DB to see if the document exists with that org scope
4. Check if the document's `dealId` → `deal.organizationId` chain resolves correctly

**Fix pattern:** Ensure the batch delete handler resolves org scope the same way the document list endpoint does (since the list works — you can see the documents).

**Test:** Select a document in the UI → Delete → should succeed without 404.
