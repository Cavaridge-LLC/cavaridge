Upgrade MERIDIAN's authentication system for proper multi-tenant support.

REPLACE basic Replit Auth with a more robust system. Use one of these approaches (in order of preference — use whichever is easiest to implement in Replit):

OPTION A — NextAuth.js (preferred):
Install and configure NextAuth.js (next-auth) with:
1. Email/password credentials provider:
   - Registration page at /register
   - Login page at /login
   - Password hashing with bcrypt
   - Add "password_hash" column to users table
   
2. Session management:
   - JWT-based sessions (stored in httpOnly cookie)
   - Session includes: userId, organizationId, role, name, email
   - Session expires after 24 hours of inactivity
   - "Remember me" option extends to 30 days

3. Protected routes:
   - All app routes require authentication
   - Unauthenticated users redirect to /login
   - API routes return 401 if no valid session

OPTION B — If NextAuth.js is too complex, use a simpler custom approach:
1. Login page at /login with email + password
2. bcrypt for password hashing
3. JWT token stored in httpOnly cookie
4. Middleware that checks the cookie on every request
5. /api/auth/login, /api/auth/logout, /api/auth/me endpoints
