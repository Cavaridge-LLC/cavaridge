ignored as evidence. Fix this so every image is analyzed, described, classified, 
and chunked for RAG search just like PDFs and text files.

Use this exact prompt when sending images to the vision model:

"""
You are an IT due diligence analyst reviewing evidence for a mergers & 
acquisitions assessment. Analyze this image and provide:

1. DESCRIPTION: A detailed description of what this image shows. Be specific 
   about any technology, infrastructure, configurations, dashboards, network 
   diagrams, error messages, or security-relevant content visible.

2. CLASSIFICATION: Classify this image into exactly ONE of these categories:
   - "Network Documentation" (network diagrams, topology maps, VLAN configs)
   - "Security Assessment" (vulnerability scans, pentest results, security dashboards)
   - "Asset Inventory" (hardware photos, rack diagrams, equipment lists)
   - "Compliance Documentation" (compliance dashboards, audit results, certificate screenshots)
   - "IT Policy" (policy documents, configuration screenshots, settings pages)
   - "Identity & Access" (Active Directory screenshots, SSO configs, user management)
   - "Infrastructure & Architecture" (server configs, cloud dashboards, monitoring)
   - "Backup & DR" (backup job screenshots, DR configs, replication status)
   - "Clinical Systems" (EHR/EMR screenshots, medical system interfaces)
   - "OT/ICS Systems" (SCADA, PLC, manufacturing system screens)
   - "IT Financial" (licensing dashboards, cost reports, billing screenshots)
   - "Organization & Staffing" (org charts, team structure diagrams)
   - "Vendor Contract" (contract screenshots, portal dashboards)
   - "Application Inventory" (software lists, SaaS dashboards, app catalogs)
   - "Unclassified" (only if nothing above fits)

3. FINDINGS: List any IT due diligence findings visible in this image. For 
   each finding, state:
   - What you observed
   - Why it matters for M&A risk assessment
   - Severity: critical, high, medium, or low
   
   Examples of what to look for:
   - End-of-life software versions visible
   - Security warnings or failed scans
   - Single points of failure in network diagrams
   - Misconfigured settings
   - Compliance gaps or expired certificates
   - Outdated hardware visible in rack photos
   - Unlicensed or shadow IT visible
   - Poor network segmentation in topology diagrams
   
   If no findings are apparent, state "No actionable findings identified."

4. EXTRACTED TEXT: Transcribe any readable text in the image (server names, 
   IP addresses, software versions, error messages, configuration values, 
   labels, etc.). This text will be used for keyword search.

Respond in this exact JSON format:
{
  "description": "...",
  "classification": "...",
  "findings": [
    {
      "observation": "...",
      "risk_relevance": "...",
      "severity": "critical|high|medium|low"
    }
  ],
  "extracted_text": "..."
}

If the image is blurry, corrupted, or not IT-related, still provide your best 
assessment. Set classification to "Unclassified" if the content is not relevant 
to IT due diligence.
"""

Update the document processing pipeline (Step 3 — Text Extraction) to handle

  Images → extracted_text = "[Image file — visual analysis pending]"
  → extraction_status = "stored"
  → classification = "Unclassified"
  → No chunks created
  → Invisible to RAG and search

NEW BEHAVIOR:
  Images → send to analyzeImage() function
  → Parse the JSON response
  → Set extracted_text = response.description + "\n\n" +

  → Set classification = response.classification
  → Set extraction_status = "extracted"
  → Store response.findings in metadata_json under key "vision_findings"
  → Store the full vision response in metadata_json under key "vision_analysis"
  → Proceed to Step 5 (chunking) — chunk the description + extracted text 
    so it's searchable via RAG
  → Update documents_analyzed count on the deal

1. Process images ONE AT A TIME with 1-second delays between calls
2. Skip images larger than 20MB (set status to "skipped_too_large")

   - metric: "vision_analyses"
   - Increment per image analyzed
5. If on Starter plan: limit to 50 image analyses per month
   If on Professional plan: limit to 500 per month
   If on Enterprise plan: unlimited
6. Resize images larger than 4MB before sending to the API:
   - Use the "sharp" npm package (install it)
   - Resize to max 2048px on longest side, maintain aspect ratio
   - Convert to JPEG at 85% quality to reduce payload size
   - This keeps API costs down without losing meaningful detail
