## 1. Overview

The AEGIS Identity Access Review (IAR) module provides automated Microsoft 365 tenant user security analysis for MSPs and their clients. The module ingests standard M365 admin center exports, correlates user identity data with activity telemetry, classifies risk, and generates a branded deliverable-grade report.

The module operates in two tiers: a public freemium landing page for one-time analysis (lead generation), and a full tenant-integrated agent within the AEGIS platform for recurring quarterly access reviews with historical diffing and remediation workflows.

### 1.1 Strategic Positioning

- **Freemium Tier:** Public-access, no-login identity analysis — mirrors the AEGIS freemium scan landing page model
- **AEGIS Upsell:** Recurring automated reviews, historical trend analysis, remediation task generation, ConnectSecure integration
- **Astra Cross-Sell:** Client-facing vCIO reporting, license optimization dashboards, executive-ready posture summaries delivered through the Astra interface
- **Cavalier Partners Channel:** Packaged as a partner-deliverable tool via CVG-CAVALIER for channel MSPs

### 1.2 Competitive Landscape

No MSP-focused tool currently provides a free, self-service M365 identity security analysis with a single-upload workflow. Existing approaches require full tenant integration (AdminDroid, CoreView) or manual spreadsheet work. The IAR module's zero-integration freemium entry point is a unique acquisition channel.

## 2. Freemium Tier — Public Landing Page

### 2.1 User Flow

1. User lands on public page (e.g., `aegis.cavaridge.com/identity-review` or `identity.cavaridge.com`)
2. Lead capture: Name, email, company name, MSP affiliation (optional)
3. Guided export instructions: step-by-step walkthrough with screenshots showing how to pull both CSV exports from the M365 Admin Center
4. File upload: Two CSV files (Entra ID user export + M365 Active User Detail report)
5. Processing: Client-side validation → server-side correlation → risk classification
6. Report delivery: Branded XLSX download (Cavaridge-branded, not tenant-branded)
7. Upsell CTAs: "Automate this quarterly with AEGIS" + "Get executive dashboards with Astra"

### 2.2 Technical Architecture

- **Frontend:** Static page hosted on Railway, React + Tailwind, no authentication required
- **File Processing:** Express 5 route accepting multipart uploads, temporary in-memory processing only
- **No Data Retention:** Uploaded CSVs and generated reports are processed in-memory and discarded after delivery. No tenant data is stored on the freemium tier. Privacy statement displayed prominently.
- **Rate Limiting:** IP-based rate limiting (3 reports/hour/IP) to prevent abuse
- **Lead Capture:** Email + metadata stored in Cavaridge CRM pipeline (Astra lead source tag)

### 2.3 Report Output (Freemium)

The freemium report mirrors the structure built in the initial prototype, with five tabs:

- **Executive Summary** — metrics table, risk counts, key recommendations
- **All Users** — full user inventory with type classification, block status, license status, activity dates, risk flags
- **Flagged Users** — isolated view of accounts with computed risk flags
- **External Guests** — dedicated guest account analysis with account age and staleness indicators
- **License Breakdown** — per-user license allocation and service-level activity

Freemium reports carry Cavaridge branding and a footer: "Generated by AEGIS Identity Review — Powered by Ducky Intelligence."

### 2.4 Astra Cross-Sell Hook

The freemium report includes a final tab or cover sheet titled "What's Next?" that positions two upgrade paths:

- **AEGIS:** "Automate quarterly identity reviews, get historical trending, and generate remediation tickets directly in your PSA."
- **Astra:** "Deliver executive-ready security posture reports to your clients with branded dashboards, license cost analysis, and board-level risk summaries."

The Astra hook is critical: the data flowing through IAR (license counts, user posture, activity trends) is exactly the data Astra needs for vCIO reporting. The freemium report is a proof-of-concept for the Astra deliverable.

## 3. Full Tier — AEGIS Identity Review Agent

### 3.1 Agent Architecture

The IAR agent operates within the AEGIS agent-first architecture as a Layer 3 product agent, consuming Layer 1 domain agents (Cybersecurity, Data Privacy) for contextual enrichment.

#### Agent Flow

1. MSP tech selects client tenant in AEGIS
2. Agent prompts for data source: guided CSV upload OR direct Microsoft Graph API pull (if tenant is Graph-connected)
3. Ingestion: UPN-based correlation, user type classification, risk flag computation
4. Delta Analysis: Compare against previous review snapshot (if available)
5. Report Generation: Tenant-branded XLSX + natural language executive summary
6. Remediation Actions (optional): Generate disable/revoke tasks, push to PSA/ConnectSecure
7. Archive: Store snapshot for next review cycle

### 3.2 Microsoft Graph API Integration (Phase 2)

For tenants with delegated Graph permissions, the agent can pull user and activity data directly, eliminating the CSV export step entirely. This requires:

- Delegated permissions: `User.Read.All`, `Reports.Read.All`, `AuditLog.Read.All`
- Tenant consent via the Cavaridge connector framework (Doppler-managed credentials)
- Scheduled pull: Agent can auto-run on a configurable cadence (monthly/quarterly)

### 3.3 Historical Diffing Engine

Each completed review stores a normalized snapshot per tenant. Subsequent reviews generate a delta report showing:

- New users added since last review
- Users removed or deleted
- License changes (assigned, revoked, upgraded, downgraded)
- Block status changes
- Activity changes (users who went inactive, users who became active)
- Risk flag changes (new flags, resolved flags)

Delta data is surfaced as a dedicated "Changes Since Last Review" tab in the report and as a natural language summary in the executive section.

### 3.4 Astra Integration

The full-tier IAR data feeds directly into Astra for client-facing delivery:

- **Astra Dashboard Widget:** Identity posture score, license utilization gauge, guest account trend chart
- **Astra vCIO Report Section:** Auto-generated "Identity & Access Review" section for quarterly business reviews, pulling IAR data without manual re-entry
- **Astra License Optimization:** IAR's license waste detection (blocked+licensed, inactive+licensed) feeds Astra's cost analysis module for projected savings calculations
- **Astra Executive Summary:** Board-level risk narrative generated from IAR findings, suitable for client-facing delivery without MSP jargon

## 4. Data Model

### 4.1 Identity Review Snapshot

Stored per-tenant per-review in the Cavaridge platform Supabase instance. RLS enforced at the tenant level via UTM.

| Table | Description | Key Fields |
|---|---|---|
| `identity_reviews` | Review metadata per tenant per run | `id`, `tenant_id`, `review_date`, `source_type` (csv\|graph), `status`, `created_by` |
| `identity_review_users` | Normalized user records per review | `id`, `review_id`, `upn`, `display_name`, `user_type`, `is_blocked`, `is_licensed`, `license_sku`, `last_activity_date`, `risk_flags[]` |
| `identity_review_deltas` | Computed changes between consecutive reviews | `id`, `review_id`, `previous_review_id`, `upn`, `change_type` (added\|removed\|modified), `field_changed`, `old_value`, `new_value` |
| `identity_review_reports` | Generated report artifacts | `id`, `review_id`, `format` (xlsx\|pdf), `storage_path`, `generated_at` |

### 4.2 Risk Flag Taxonomy

| Flag | Severity | Condition | Recommended Action |
|---|---|---|---|
| Blocked but Licensed | High | Sign-in blocked = true AND license assigned | Revoke license immediately |
| External with License | High | User type = External Guest AND license assigned | Review justification; revoke if unjustified |
| Inactive Licensed (>90d) | Medium | Licensed AND last activity >90 days ago | Confirm with stakeholder; consider revocation |
| Inactive Licensed (>180d) | High | Licensed AND last activity >180 days ago | Strong reclamation candidate |
| Licensed — No Activity Data | Medium | Licensed AND no matching O365 activity record | Investigate; may be service/room account |
| Stale External Guest | Low | External, unlicensed, unblocked, account >180 days | Disable or delete |
| Password Never Expires | Medium | Password never expires = true (member accounts) | Enforce password policy or transition to passwordless |
| No MFA Registered | High | Member account with no MFA methods (Graph only) | Enforce MFA registration |

## 5. Processing Pipeline

### 5.1 Ingestion

- CSV Parsing: papaparse (TypeScript) or pandas (Python microservice) with BOM handling, encoding detection, and header normalization
- UPN Normalization: lowercase, trim, strip mailto: prefixes
- User Type Classification: `#EXT#` detection for external guests; all others classified as Member
- Date Parsing: ISO 8601 with UTC normalization; handle blank/missing dates gracefully

### 5.2 Correlation

- Primary join key: UPN (case-insensitive)
- Left join from Entra ID users to O365 activity (users without activity records are flagged, not dropped)
- Activity date aggregation: most recent date across Exchange, OneDrive, SharePoint, Teams, Yammer, Skype

### 5.3 Risk Classification

- Deterministic rule engine — no LLM required for flag computation
- Each user evaluated against the full risk flag taxonomy
- Multiple flags per user supported (semicolon-delimited in output)
- Severity rollup: highest severity flag determines the user's row highlight color in the report

### 5.4 Report Generation

- XLSX generation via openpyxl (Python) or ExcelJS (TypeScript)
- Cavaridge DIT brand standards: Arial, H1 #2E5090, table headers blue/white, #F2F6FA row banding, #BFBFBF borders
- Tenant branding applied on full-tier reports (logo, company name in header)
- Freemium reports carry Cavaridge branding only

### 5.5 Executive Summary Generation (Full Tier)

- Natural language narrative generated by Ducky Intelligence (Spaniel LLM gateway)
- Prompt template includes: metric counts, risk flag distribution, delta summary (if available), client context from tenant profile
- Output: 3–5 paragraph executive summary suitable for client delivery without MSP jargon
- Reviewed by MSP tech before inclusion in final report

## 8. Security & Compliance

- **Freemium tier:** No data retention. CSVs processed in-memory and discarded. No tenant data stored.
- **Full tier:** All data tenant-scoped via UTM and Supabase RLS. No cross-tenant data leakage.
- **Graph credentials:** Stored in Doppler, never in code or environment variables.
- **Report artifacts:** Stored in Supabase Storage with tenant-scoped access policies.
- **SOC 2 alignment:** Audit logging on all review actions (who ran the review, when, what was generated).
- **HIPAA consideration:** IAR processes identity metadata only (names, emails, license status). No PHI is ingested. However, user lists from healthcare tenants are treated as sensitive and subject to the same RLS protections as all tenant data.
